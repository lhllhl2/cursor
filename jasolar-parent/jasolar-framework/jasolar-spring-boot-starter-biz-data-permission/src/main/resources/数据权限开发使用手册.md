# 数据权限开发使用手册
## 1. 概述
   本手册旨在指导开发者如何使用数据权限模块来控制各个业务场景下的数据权限操作
   
## 2.主要接口
### 2.1 IUserScopeDO
> 用于定义与用户相关的数据权限 
>
> ```java  
> public interface IUserScopeDO {
>    /**
>     * @return 人员工号
>     */
>    String getUserNo();
>
>    /**
>     * 设置 人员工号
>     *
>     * @param userNo 人员工号
>     */
>    BaseUserScope setUserNo(String userNo);
> } 
> ```

### 2.2 ILegalScopeDO
> 用于定义与法人相关的数据权限 
>
> ```java  
>  public interface ILegalScopeDO {
>
>    /**
>     * @return 法人编码
>     */
>    String getLegalCode();
>
>    /**
>     * 设置法人编码
>     * 
>     * @param legalCode 法人编码
>     */
>    BaseLegalScope setLegalCode(String legalCode); 
> }
> ```


### 2.3 IDeptScopeDO
> 用于定义与部门相关的数据权限
>
> ```java   
>  public interface IDeptScopeDO {
>    /**
>     * @return 部门CODE
>     */
>    String getDeptCode();
>
>    /**
>     * 设置 部门CODE
>     * 
>     * @param deptId 部门CODE
>     */
>    BaseDeptScope setDeptCode(String deptCode); 
> }
> ```

## 3. 实现数据权限接口 
### 3.1 实现 BaseUserScopeDO
> 实体类具备用户权限控制功能，需要实现 BaseUserScopeDO 接口。在实体类中实现获取和设置人员工号的逻辑。
> ```java 
> public class UserEntity implements BaseUserScopeDO {  
>     private String userNo;  
> 
>     @Override  
>     public String getUserNo() {  
>         return userNo;  
>     }  
> 
>     @Override  
>     public BaseUserScope setUserNo(String userNo) {  
>         this.userNo = userNo;  
>         return this;  
>     }  
> }  
> ```

### 3.2 实现 BaseLegalScopeDO
> 同样，实现 BaseLegalScopeDO接口 来控制法人数据权限。
> ```java 
> public class LegalEntity implements BaseLegalScopeDO {  
>     private String legalCode;  
> 
>     @Override  
>     public String getLegalCode() {  
>         return legalCode;  
>     }  
> 
>     @Override  
>     public BaseLegalScope setLegalCode(String legalCode) {  
>         this.legalCode = legalCode;  
>         return this;  
>     }  
> }   
> ```

### 3.3 实现 BaseDeptScopeDO
> 如果需要控制部门数据权限，可以实现 BaseDeptScopeDO 接口。
> ```java 
> public class DeptEntity implements BaseDeptScopeDO {  
>     private String deptCode;  
> 
>     @Override  
>     public String getDeptCode() {  
>         return deptCode;  
>     }  
> 
>     @Override  
>     public BaseDeptScope setDeptCode(String deptCode) {  
>         this.deptCode = deptCode;  
>         return this;  
>     }  
> }  
> ```

## 4. 使用 DataPermission,EnableDataPermission 注解
### 4.1 添加DataPermission注解

> 在需要进行数据权限控制的接口或类上，添加 DataPermission 注解。此注解会自动将用户的权限信息应用于 SQL 查询中。
> ```java
> 
> import com.example.annotations.DataPermission;
>
> @DataPermission  
> @RestController  
> @RequestMapping("/data")  
> public class DataController {
>
>     @GetMapping("/list")  
>     public List<DataEntity> getList() {  
>          
>     }  
> }   
> ```

### 4.2 EnableDataPermission注解

> 在不需要进行数据权限控制的接口或类上，添加 EnableDataPermission 注解。此注解在value为false的情况下，会禁用该接口的请求链路上的所有权限逻辑. 只有当外部有此注解为false的情况下,才需要在内层调用的方法上显示添加value=true的注解,使用DataPermission注解默认就会开启权限
> ```java
>  
>
> @EnableDataPermission(false)  
> @RestController  
> @RequestMapping("/data")  
> public class DataController {
>
>     @GetMapping("/list")  
>     public List<DataEntity> getList() {  
>          
>     }  
> }  
> ```
### 4.3 数据权限过滤
在请求到达控制器后，权限框架会根据当前用户的权限信息，自动修改 SQL 查询，添加相应的 SQL 条件。<br>
这些条件字段包括：<br>
1、userNo: 当前用户的员工号 <br>
2、legalCode: 当前用户权限范围内的法人编码 <br>
3、deptCode: 当前用户权限范围内的部门 CODE <br>

## 5. 重要提示
   1、确保在控制器或者服务类上使@DataPermission 注解，并且实体类需要正确实现相关接口后，才能触发数据权限的自动过滤。


## 6.特殊情况下如何实现的数据权限控制
### 6.1 多表查询,使用其中一个表作为数据权限控制 字段:OR连接,维度:OR连接
描述： 需要查询数据表 table1,table2,需要对table2实现多维度，多字段的数据权限控制,并且维度和字段均通过OR连接
>sql如下  
>```sql
> SELECT * FROM table1 a
> INNER JOIN table2 b ON a.table2_id = b.id
> WHERE ( (b.user_no = 'xxx' OR b.creator = 'xxx') OR b.legal_code IN ('aaa, 'ccc'))
> ```
>注解对应如下
> ```java 
> @DataPermission( {
>     @DataRule( value = UserDataPermissionRule.class, tables = { alias = "b", coulumns = { "user_no", "creator" } } ) },
>     @DataRule( value = LegalDataPermissionRule.class, tables = { alias = "b", coulumns = { "legal_code" } } ) } )
> )
> ```

### 6.2 多表查询,使用不同的表来控制多字段和不同的权限维度,字段:OR连接,维度:AND连接
描述： 需要查询数据表 table1,table2,需要对table2实现多维度，多字段的数据权限控制,并且维度之间通过AND连接，字段通过OR连接
>sql如下
>```sql
> SELECT * FROM table1 a
> INNER JOIN table2 b ON a.table2_id = b.id
> WHERE ( (a.user_no = 'xxx' OR a.creator = 'xxx') AND b.legal_code IN ('aaa, 'ccc'))
>```

>注解对应如下
> ```java 
> @DataPermission( @DataRule( value = UserDataPermissionRule.class, tables = { alias = "a", coulumns = { "user_no", "creator" } } ) )
> @DataPermission( @DataRule( value = LegalDataPermissionRule.class, tables = { alias = "b", coulumns = { "legal_code" } } )  )
> ```
 
### 6.3. 单表查询，控制不同的权限维度以及多字段的数据权限,维度:AND连接,字段:OR连接

>sql如下
>```sql
>    SELECT a.user_no, a.creator, a.legal_code FROM table1 a
>    WHERE ( (a.user_no = 'xxx' OR a.creator = 'xxx') AND a.legal_code IN ('aaa, 'ccc'))
>```

>注解对应如下
> ```java 
> @DataPermission( @DataRule( value = UserDataPermissionRule.class, tables = { alias = "a", coulumns = { "user_no", "creator" } } ) )
> @DataPermission( @DataRule( value = LegalDataPermissionRule.class, tables = { alias = "a", coulumns = { "legal_code" } } )  )
> ```
 
### 6.4 多表查询，使用不同的表来控制不同的权限维度,维度:OR,字段:OR,需要改写SQL
>sql如下
>```sql
> SELECT * FROM (
> 	SELECT a.user_no, a.creator, b.legal_code FROM table1 a
> 	INNER JOIN table2 b ON a.a = b.b
> ) x
> WHERE ( (x.user_no = 'xxx' OR x.creator = 'xxx') OR x.legal_code IN ('aaa, 'ccc'))
>```

>注解对应如下
> ```java 
>    @DataPermission({
> 	   @DataRule( value = UserDataPermissionRule.class, tables = { alias = "x", coulumns = { "user_no", "creator" } } ) ,
> 	   @DataRule( value = LegalDataPermissionRule.class, tables = { alias = "x", coulumns = { "legal_code" } } ) 
>    })
> ```

 ## 7.关于读写分离的使用

> 在使用 @DataPermission 注解时，可以根据具体的业务需求选择合适的 ReadWrite 类型。例如：
> 
> READ指该接口可以获取对应权限范围的读权限
> ```java 
> @DataPermission(readWrite = ReadWrite.READ)  
> public void viewData() {  
>      
> }  
> ```
>
> WRITE指该接口可以获取对应权限范围的写权限
> ```java 
> @DataPermission(readWrite = ReadWrite.WRITE)  
> public void updateData() {  
>      
> }  
> ```
>
> ALL指允许对数据进行所有操作，包括权限范围的读和写
> ```java 
> @DataPermission(readWrite = ReadWrite.ALL)  
> public void viewData() {  
>      
> }  
> ```
